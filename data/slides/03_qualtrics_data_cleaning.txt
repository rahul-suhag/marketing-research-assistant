QUALTRICS DATA CLEANING AND PREPROCESSING GUIDE
=================================================

1. UNDERSTANDING QUALTRICS CSV EXPORTS
----------------------------------------

When you export data from Qualtrics as a CSV, the file typically contains:

Row 1: Column headers (short codes like Q1, Q2, Q4#1_1)
Row 2: Full question text (the actual survey questions as displayed to respondents)
Row 3: Import IDs (JSON-formatted Qualtrics internal identifiers like {"ImportId":"QID1_TEXT"})
Row 4 onwards: Actual survey responses

Metadata Columns (automatically generated by Qualtrics):
- StartDate: When the respondent started the survey
- EndDate: When the respondent finished the survey
- Status: Response status (0 = normal, 1 = preview, 2 = test)
- IPAddress: Respondent's IP address
- Progress: Percentage of survey completed (0-100)
- Duration (in seconds): Time spent on the survey
- Finished: Whether the survey was completed (1 = yes, 0 = no)
- RecordedDate: When the response was recorded
- ResponseId: Unique identifier for each response (e.g., R_abc123)
- RecipientLastName: Last name (if distributed via contact list)
- RecipientFirstName: First name (if distributed via contact list)
- RecipientEmail: Email (if distributed via contact list)
- ExternalReference: External data reference
- LocationLatitude: Approximate latitude of respondent
- LocationLongitude: Approximate longitude of respondent
- DistributionChannel: How the survey was distributed (anonymous, email, etc.)
- UserLanguage: Language the survey was displayed in


2. STEP-BY-STEP DATA CLEANING PROCESS
---------------------------------------

Step 1: Remove Metadata Rows (Row 2 and Row 3)

In Excel (Windows):
1. Open the CSV file in Excel.
2. Click on row number 2 on the left side to select the entire row.
3. Hold Shift and click on row number 3 to also select it.
4. Right-click on the selected rows and choose Delete.
5. Your data now starts with actual responses in row 2.

In Excel (Mac):
1. Open the CSV file in Excel.
2. Click on row number 2 on the left side to select the entire row.
3. Hold Shift and click on row number 3 to also select it.
4. Right-click (or Control+click) on the selected rows and choose Delete.
5. Same result as Windows.

In Python/pandas:
df = pd.read_csv('survey_data.csv')
df = df.iloc[2:].reset_index(drop=True)  # Skip the question text and import ID rows


Step 2: Remove Unnecessary Metadata Columns

Columns to consider removing for analysis:
- IPAddress (privacy; not useful for most analyses)
- RecipientLastName, RecipientFirstName, RecipientEmail (privacy)
- ExternalReference (usually empty)
- LocationLatitude, LocationLongitude (unless geographic analysis is needed)
- RecordedDate (redundant with EndDate)

In Excel:
1. Right-click on the column header letter of the column to delete.
2. Select Delete.
3. Repeat for each unnecessary column.

Keep these metadata columns (useful for data quality):
- StartDate, EndDate
- Progress
- Duration (in seconds)
- Finished
- ResponseId
- DistributionChannel


Step 3: Filter Out Incomplete Responses

Method A: Filter by Finished column
- In Excel: Data > Filter > Click the dropdown on the Finished column > Uncheck 0 > OK.
- This shows only completed responses.
- Copy the filtered data to a new sheet for analysis.

Method B: Filter by Progress column
- Filter for Progress = 100 to keep only responses where the entire survey was completed.

Method C: Remove previews and test responses
- Filter the Status column: Keep only Status = 0 (normal responses).
- Remove Status = 1 (preview) and Status = 2 (test).


Step 4: Check Response Duration for Speeders

Respondents who finish too quickly may not have read the questions carefully.

Rule of thumb: Remove responses where Duration < (number of questions * 2 seconds).

In Excel:
1. Sort by the Duration (in seconds) column: Data > Sort > Sort by Duration > Smallest to Largest.
2. Review the shortest durations.
3. Delete rows with unreasonably short durations.
4. Also check for unreasonably LONG durations (respondent may have left the tab open).

Alternative: Calculate the median duration and flag responses below 1/3 of the median.
Formula: =IF(E2 < MEDIAN($E$2:$E$100)/3, "SPEEDER", "OK")


Step 5: Handle Missing Values

Identifying Missing Values in Excel:
- Use COUNTBLANK: =COUNTBLANK(B2:B100) to count blanks in a column.
- Use Conditional Formatting to highlight blank cells:
  Windows: Home > Conditional Formatting > Highlight Cell Rules > Equal to > (leave blank) > Format.
  Mac: Home > Conditional Formatting > same process.

Strategies for Missing Data:
a) Listwise Deletion: Remove the entire row if ANY response is missing.
   - Conservative approach; reduces sample size.
   - Appropriate when missing data is random (MCAR) and sample is large enough.

b) Pairwise Deletion: Include the case in any analysis where it has complete data.
   - Preserves more data but can lead to inconsistent sample sizes across analyses.

c) Mean/Median Imputation: Replace missing values with the variable's mean or median.
   - In Excel: =IF(ISBLANK(B2), AVERAGE($B$2:$B$100), B2)
   - Simple but can reduce variance and distort relationships.

d) Leave as Missing: For open-ended questions, missing is often valid (respondent chose not to answer).


Step 6: Recode Variables

Binary Coding (for Yes/No or two-category variables):
- =IF(A2="Yes", 1, 0)
- =IF(A2="Male", 1, 0)

Reverse Coding (for negatively worded Likert items on a 1-5 scale):
- =6-A2  (this converts 1 to 5, 2 to 4, 3 stays 3, 4 to 2, 5 to 1)
- General formula for k-point scale: =(k+1)-A2

Recoding Likert Text to Numbers:
- =IF(A2="Strongly Disagree",1,IF(A2="Disagree",2,IF(A2="Neutral",3,IF(A2="Agree",4,IF(A2="Strongly Agree",5,"")))))
- Or use SWITCH (Excel 2019+): =SWITCH(A2,"Strongly Disagree",1,"Disagree",2,"Neutral",3,"Agree",4,"Strongly Agree",5)

Creating Dummy Variables from Categorical Data:
For a variable "Region" with values North, South, East, West:
- Dummy_South: =IF(A2="South",1,0)
- Dummy_East: =IF(A2="East",1,0)
- Dummy_West: =IF(A2="West",1,0)
- North is the reference category (omitted)
- Include all dummy variables EXCEPT the reference in regression


Step 7: Create Composite Variables (Summated Scales)

When multiple Likert items measure the same construct, combine them into a composite scale.

Steps in Excel:
1. First, reverse-code any negatively worded items (see Step 6).
2. Create a new column with the average of the items:
   =AVERAGE(B2, C2, D2, E2)
   This computes the mean of 4 Likert items for each respondent.
3. Check reliability: The Cronbach's alpha should be 0.70 or higher.

Calculating Cronbach's Alpha in Excel (Manual Method):
1. Compute the variance of each item column: =VAR.S(B2:B100)
2. Compute the variance of the sum/total score column: =VAR.S(SUM_COL)
3. Apply the formula: alpha = (k/(k-1)) * (1 - sum_of_item_variances / total_variance)
   Where k = number of items.
4. Excel formula: =(k/(k-1))*(1-(sum_item_var/total_var))


Step 8: Data Validation and Final Checks

Before analysis, verify:
1. All variables are in the correct format (numbers are numbers, not text).
   - Select a data column > Check if numbers are left-aligned (text) or right-aligned (numbers).
   - If numbers are stored as text: Select the column > Data > Text to Columns > Finish.
   - Or: Multiply by 1: In a new column, =A2*1 to force conversion.

2. No out-of-range values:
   - For 1-5 Likert scales: =IF(OR(A2<1, A2>5), "OUT OF RANGE", "OK")
   - For age: =IF(OR(A2<18, A2>100), "CHECK", "OK")

3. Check for duplicate responses:
   - Sort by ResponseId and check for duplicates.
   - Or use: =COUNTIF($A$2:$A$100, A2) and flag any value > 1.

4. Verify sample size meets minimum requirements:
   - For t-tests: Minimum 30 per group (recommended).
   - For ANOVA: Minimum 20 per group.
   - For regression: Minimum 10-20 observations per predictor variable.
   - For factor analysis: Minimum 5-10 observations per variable, or at least 100 total.


3. CLEANING SPECIFIC QUALTRICS QUESTION TYPES
-----------------------------------------------

Matrix Questions (e.g., Q4#1_1, Q4#1_2):
- Each sub-question becomes a separate column.
- Naming convention: Q[number]#[row]_[column]
- These are typically ordinal (Likert-type).
- Clean by converting to numeric and checking ranges.

Multiple Choice Questions:
- Single answer: One column with the selected option text.
- Multiple answer: Separate columns for each option, coded 1 (selected) or blank/0 (not selected).
- Recode text responses to numeric codes for analysis.

Text Entry Questions (Open-Ended):
- Do not convert to numeric.
- Can be used for qualitative analysis (coding themes).
- Check for blank/NA responses.
- These are nominal-level data.

Slider Questions:
- Already numeric (0-100 typically).
- Ratio scale data.
- Check for the default value (usually 50) which may indicate non-response.

Rank Order Questions:
- Each ranked item gets a column with the rank number.
- Ordinal scale data.
- Handle ties and missing ranks.

Side-by-Side Matrix Questions:
- Create multiple columns, one for each rating dimension per group.
- Example: Q4#1_1 = Dimension 1 for Group 1, Q4#1_2 = Dimension 1 for Group 2.
- Each column is typically ordinal (rating scale).
